# Где размещен хостинг сайта

## Архитектура

✅ **Хостинг сайта находится на вашем сервере** `212.74.227.208`

### Компоненты:

1. **Домен (ispravleno.pro)** - регистратор: reg.ru
   - Используется только для управления доменом и DNS записями
   - DNS записи должны указывать на ваш сервер: `212.74.227.208`

2. **Хостинг (сайт)** - ваш сервер: `212.74.227.208`
   - Next.js приложение работает на порту 3003
   - Nginx проксирует запросы с порта 80/443 на порт 3003
   - Все файлы и база данных находятся на вашем сервере

3. **Регистратор (reg.ru)** - используется только для:
   - Управления доменом
   - Настройки DNS записей
   - НЕ для хостинга сайта

## Как это работает

```
Пользователь → ispravleno.pro (DNS запись) → 212.74.227.208 (ваш сервер) → Nginx → Next.js (порт 3003)
```

1. Пользователь открывает `ispravleno.pro` в браузере
2. DNS запись указывает на `212.74.227.208`
3. Запрос приходит на ваш сервер
4. Nginx (порт 80/443) получает запрос
5. Nginx проксирует запрос на Next.js приложение (порт 3003)
6. Next.js обрабатывает запрос и возвращает страницу

## Что нужно сделать

### 1. Настроить DNS записи в reg.ru

В панели управления reg.ru нужно изменить A-записи, чтобы они указывали на ваш IP:
- `@` → `212.74.227.208`
- `www` → `212.74.227.208`

### 2. Настроить Nginx на вашем сервере

Установить и настроить Nginx для проксирования запросов:
```bash
./setup-ispravleno-pro.sh
```

### 3. Запустить Next.js приложение

Приложение должно работать на порту 3003:
```bash
pm2 restart ispravleno-website
```
